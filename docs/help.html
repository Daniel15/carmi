Help

This project is still in it's infancy if you encounter a bug, please submit an issue.

A very convenient way to inspect your deriviations is to add a tap method that calls debugger/console.log method to your function library, and .call('tap') in the part that is tricky.

const instance = modelFunction(initialState, {
  tap: val => {
    debugger;
    return val;
  }
});

If you derived state is incorrect and everything is working when using the Simple compiler it is either a bug in CARMI or your state/deriviations were mutated not by using the setters in the model, always treat your deriviations as readonly. (A good way to enforce this is to use es6 proxies to make the model readonly in debug mode)
